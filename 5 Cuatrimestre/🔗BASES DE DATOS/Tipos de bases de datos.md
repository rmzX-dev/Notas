#### Investiga los tipos de bases de datos NoSQL: 
(Documental, clave-valor, Orientada a grafos, Orientadas a objetos, Multivalor, Tabulares y de arreglos).

- Características
- Ventajas y desventajas
- Ejemplos 

| Bases de datos:     | Características                                                                                                                                                                | Ventajas                                                                                                                                                                      | Desventajas                                                                                                                         | Ejemplos                                           |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------- |
| Documentales        | - Almacenan datos en documentos similares a JSON o BSON (clave-valor anidado).<br>- No requieren esquemas estrictos.<br>- Los datos se organizan en colecciones, no en tablas. | - Flexible y compatible con cambios en los datos.<br>- Ideal para aplicaciones que requieren almacenamiento jerárquico.<br>- Consultas complejas gracias a índices avanzados. | - Menos eficientes en operaciones transaccionales complejas.<br>- No son ideales para relaciones muy profundas entre datos.         | - MongoDB<br>- CouchDB<br>- RavenDB                |
| Clave-Valor         | - Almacenan datos como un par único de clave y valor.<br>- Extremadamente rápidas debido a su simplicidad.<br>- Diseñadas para búsquedas rápidas por clave.                    | - Gran rendimiento en aplicaciones que necesitan alta velocidad.<br>- Fácil de escalar horizontalmente.                                                                       | - No soportan consultas complejas.<br>- Dificultad para representar relaciones entre datos.                                         | - Redis<br>-DynamoDB <br>- Riak                    |
| Orientados a grafos | - Diseñadas para modelar datos con relaciones complejas, utilizando nodos, aristas y propiedades.<br>- Optimizadas para consultas de grafos                                    | - Ideal para análisis de redes sociales, recomendaciones, y gestión de relaciones.<br>- Manejan relaciones complejas con eficiencia.                                          | - Curva de aprendizaje más alta.<br>- Menos eficientes para datos tabulares o no relacionados.                                      | - Neo4j<br>- ArangoDB<br>- Amazon Neptune          |
| Orientada a objetos | - Los datos se almacenan como objetos, similares a los utilizados en la programación orientada a objetos.<br>- Los atributos y métodos del objeto son parte del modelo.        | - Integra bien con lenguajes de programación orientados a objetos.<br>- Reducción en la necesidad de transformaciones de datos.                                               | - Menor rendimiento en consultas ad hoc.<br>- Escalabilidad más limitada comparada con otros tipos.                                 | - db4o<br>- ObjectDB<br>- ZopeDB                   |
| Multivalor          | - Permiten que un atributo tenga múltiples valores asociados.<br>- Usadas en aplicaciones que manejan datos jerárquicos o complejos.                                           | - Más eficientes en almacenamiento que las bases de datos relacionales para ciertos modelos de datos.<br>- Soporte para estructuras complejas.                                | - Curva de aprendizaje alta.<br>-Compatibilidad limitada con otros sistemas.                                                        | - Rocket U2<br>-OpenInsight<br>- jBASE             |
| Tabulares           | - Utilizan un modelo similar a las tablas de una base de datos relacional, pero sin esquemas estrictos.<br>- Diseñadas para manejar grandes volúmenes de datos estructurados.  | - Escalabilidad horizontal.<br>- Alta velocidad en operaciones de lectura y escritura.                                                                                        | - Menos flexibles que otros tipos de bases de datos NoSQL.<br>- Operaciones de análisis complejas requieren soluciones adicionales. | - Apache Cassandra<br>- HBase<br>- Google Bigtable |
| De arreglos         | - Especializadas en almacenar y gestionar datos multidimensionales, como matrices y datos espaciales.<br>- Diseñadas para análisis de big data científico y geoespacial.       | - Excelente rendimiento en aplicaciones científicas y de análisis.<br>- Optimización para cálculos complejos.                                                                 | - Menos adecuadas para datos transaccionales o generales.<br>- Menor adopción en comparación con otros modelos.                     | - SciDB<br>- Rasdaman<br>- TileDB                  |
